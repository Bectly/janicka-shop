## Fixed in this cycle

### F1. Manager tab list hidden behind admin header on scroll
- **File**: `src/components/admin/manager/tabs-shell.tsx:90`
- **Symptom**: Admin layout header is `sticky top-0 z-30 h-14`. Manager tabs were `sticky top-0 z-10` — tabs slid under the header on scroll, leaving no way to switch tabs without scrolling back up.
- **Fix**: Tabs now stick at `top-14` (matching header height) on all viewports; dropped `md:static`/`md:bg-transparent` so tabs stay visible on desktop too.
- **Verified**: tsc clean.

## Open findings (recommended for Trace / Bolt)

### O1. Inline `<img>` in thread-message-blocks bypasses Next image pipeline
- **File**: `src/components/admin/manager/thread-message-blocks.tsx:88`
- Uses raw `<img>` (eslint-disabled) without `loading="lazy"` or `decoding="async"`. If the manager replies with multiple images, LCP on the manager page degrades. Switch to `next/image` with a known remote pattern, or at minimum add `loading="lazy" decoding="async"`.

### O2. Action/poll buttons rely on `title` for "not yet implemented" hint
- **File**: `src/components/admin/manager/thread-message-blocks.tsx:101-135`
- `disabled` buttons with `title="Akce — zatím jen zobrazení (J28)"` — `title` does not surface on touch devices. On mobile users see a greyed button with no explanation. Add a small caption ("Akce — zatím náhled, brzy aktivní") below the button row, or render a tag chip.

### O3. Pie chart label rendering at 375 px
- **File**: `src/components/admin/manager/thread-message-blocks.tsx:208-219`
- Pie chart with `outerRadius={70}` + default `label` prop renders text labels outside the slice; at 375 px viewport (~340 px chart width) labels can overflow the rounded card. Either pass a label renderer that truncates, or hide labels and rely on tooltip + legend on narrow viewports.

### O4. Composer textarea `resize-y` allows vertical-only resize, but no max-height
- **File**: `src/components/admin/manager/threads-tab.tsx:310`
- User can drag the textarea to occupy the entire viewport. Cap with `max-h-64` (or similar) to prevent the form from pushing the thread list off-screen on tablet/desktop.

### O5. Admin tables use `overflow-x-auto` without scroll affordance
- **Files**: `src/app/(admin)/admin/orders/orders-table.tsx:240`, `customers/page.tsx:303`, `subscribers/page.tsx:149`, `referrals/page.tsx:231`, etc.
- All admin tables wrap in `overflow-x-auto` (good) but on mobile there is no visual cue that the table scrolls horizontally. Consider adding a fade-out shadow on the right edge or a `→` icon on first paint.

### O6. Public PDP gallery — no findings
- `src/app/(shop)/products/[slug]/page.tsx` uses `ProductGallery` component; grid breakpoints are healthy (`grid-cols-2 → sm:grid-cols-3 → lg:grid-cols-4`). No regressions visible from code.

### O7. Checkout grid layout
- `src/app/(shop)/checkout/page.tsx:739` uses `grid grid-cols-1 gap-8 lg:grid-cols-3` with explicit `grid-cols-1` to prevent auto-min-content overflow (comment in code). Pattern is correct — mobile stacks vertically; desktop is 1/3 + 2/3. No issue.
