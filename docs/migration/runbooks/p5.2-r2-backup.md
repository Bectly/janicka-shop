# P5.2 — Nightly R2 backup (Turso dump + uploads)

**Task:** #339 — 30d daily + 12mo monthly backups to Cloudflare R2.
**Target host:** Hetzner VPS `46.224.219.3` (Debian).
**Backup bucket:** `janicka-backup` (separate from images bucket `janicka-shop-images`).
**Status:** Script + cron entry shipped. Bootstrap (rclone install + R2 creds + first upload) is the remaining manual step.

Files:
- `scripts/cron/backup-r2.sh` — backup script (idempotent, `--dry` supported).
- `docs/migration/cron/janicka-shop.cron` — cron entry at 03:00 UTC.

---

## 1. Pre-flight

1. **R2 bucket created.** At `dash.cloudflare.com` → R2 → Create bucket → `janicka-backup`, location `EEUR` (matches images bucket).
2. **R2 API token issued.** R2 → Manage R2 API Tokens → Create Token:
   - Permissions: **Object Read & Write**
   - Scope: bucket `janicka-backup` (or account-wide if multiple backups share creds).
   - Record the `Access Key ID` + `Secret Access Key` — Cloudflare shows them once.
3. **Register in JARVIS DB** (update existing `r2-janicka` entry or insert new `r2-janicka-backup`):
   ```bash
   sqlite3 ~/.claude/jarvis-gym/jarvis.db <<'SQL'
   INSERT INTO api_keys (name, category, service, key_value, endpoint, description)
   VALUES (
     'r2-janicka-backup',
     'infrastructure',
     'Cloudflare R2',
     'ACCESS_KEY_ID|SECRET_ACCESS_KEY|ACCOUNT_ID',
     'https://<account-id>.r2.cloudflarestorage.com',
     'R2 bucket janicka-backup — nightly backups (30d daily + 12mo monthly). Pipe-delimited: access_id|secret|account_id.'
   );
   SQL
   ```
4. **Telegram alert env.** Already have `telegram-bot` key in JARVIS DB. Add to `/opt/janicka-shop/.env.production`:
   ```
   BACKUP_TELEGRAM_BOT_TOKEN=<token from JARVIS DB telegram-bot>
   BACKUP_TELEGRAM_CHAT_ID=8750673812
   ```
5. **Turso CLI available as `www-data`.** Either install system-wide (`/usr/local/bin/turso`) or install in `www-data`'s home. Authenticate once: `sudo -u www-data turso auth login` or plant the token at `/var/www/.turso/settings.json`. Verify: `sudo -u www-data turso db list` shows `janicka-shop`.

---

## 2. Install rclone + configure R2 remote

On the VPS as root:

```bash
apt update && apt install -y rclone
```

Create the rclone config for `www-data` (the cron user):

```bash
install -d -o www-data -g www-data -m 0700 /var/www/.config/rclone
cat > /tmp/rclone.conf <<'EOF'
[r2]
type = s3
provider = Cloudflare
access_key_id = REPLACE_ME_ACCESS_KEY_ID
secret_access_key = REPLACE_ME_SECRET_ACCESS_KEY
endpoint = https://REPLACE_ME_ACCOUNT_ID.r2.cloudflarestorage.com
acl = private
region = auto
EOF
install -o www-data -g www-data -m 0600 /tmp/rclone.conf /var/www/.config/rclone/rclone.conf
rm /tmp/rclone.conf
```

(`www-data`'s home is `/var/www` on Debian. Adjust if different — `getent passwd www-data | cut -d: -f6`.)

Smoke-test connectivity:

```bash
sudo -u www-data rclone lsd r2:
# expected: line listing `janicka-backup` and any other buckets
sudo -u www-data rclone touch r2:janicka-backup/.probe
sudo -u www-data rclone delete r2:janicka-backup/.probe
```

## 4. First manual run

Dry-run (no uploads, no alert):
```bash
sudo -u www-data bash -c 'cd /opt/janicka-shop && ./scripts/cron/backup-r2.sh --dry'
```

Live:
```bash
sudo -u www-data bash -c 'cd /opt/janicka-shop && ./scripts/cron/backup-r2.sh'
```

Verify in R2:
```bash
sudo -u www-data rclone ls r2:janicka-backup/daily/$(date -u +%F)/
# expected: turso-janicka-shop-YYYY-MM-DD.sql.gz  (plus uploads tar if /opt/janicka-shop/uploads exists)
```

## 5. Deploy cron entry

```bash
sudo cp docs/migration/cron/janicka-shop.cron /etc/cron.d/janicka-shop
sudo chown root:root /etc/cron.d/janicka-shop
sudo chmod 644 /etc/cron.d/janicka-shop
sudo systemctl restart cron
```

Watch the next 03:00 UTC run:
```bash
tail -f /var/log/janicka-shop/cron.log | grep backup-r2
```

## 8. Restore rehearsal (Phase 5.3)

Tracked separately. Outline:
1. `rclone copy r2:janicka-backup/daily/<date>/turso-janicka-shop-<date>.sql.gz /tmp/`
2. `gunzip /tmp/turso-*.sql.gz`
3. Create throwaway DB: `turso db create janicka-restore-test --from-dump /tmp/turso-janicka-shop-<date>.sql`
4. Spot-check row counts against prod.
5. Delete rehearsal DB.

Acceptance for P5.3: restore produces a queryable DB within 15 minutes, row counts within 1% of prod at dump time.

---

## Rollback

Remove cron entry only (script is harmless when not invoked):
```bash
sudo sed -i '/backup-r2/d' /etc/cron.d/janicka-shop
sudo systemctl restart cron
```

Bucket contents stay in R2 at Cloudflare's usual per-GB pricing until manually deleted.
