## Execution

SSH to the VPS:
```bash
ssh kryxon
```

Install certbot + nginx plugin (idempotent — apt skips if present):
```bash
sudo apt update && sudo apt install -y certbot python3-certbot-nginx
```

Run certbot in nginx mode — it parses the vhost, adds a `:443` server block, reloads nginx, and writes a `:80 → :443` redirect:
```bash
sudo certbot --nginx \
  -d janicka-shop.cz \
  -d www.janicka-shop.cz \
  --non-interactive \
  --agree-tos \
  --email jkopecky666@gmail.com \
  --redirect \
  --hsts \
  --uir
```

Flags rationale:
- `--nginx` → auto-edit `/etc/nginx/sites-enabled/janicka-shop.conf`.
- `--redirect` → add 301 from HTTP to HTTPS (acceptance criterion).
- `--hsts` → add `Strict-Transport-Security` header.
- `--uir` → upgrade-insecure-requests CSP directive.

## Post-install verification

```bash
# 1. Cert on disk
sudo ls -la /etc/letsencrypt/live/janicka-shop.cz/
# expect: cert.pem  chain.pem  fullchain.pem  privkey.pem

# 2. HTTPS serves valid cert
curl -sI https://janicka-shop.cz | head -20
# expect: HTTP/2 200, strict-transport-security header

# 3. HTTP redirects to HTTPS
curl -sI http://janicka-shop.cz | grep -iE 'HTTP|location'
# expect: HTTP/1.1 301 Moved Permanently
#         location: https://janicka-shop.cz/

# 4. Cert expiry > 80 days (fresh cert = 90d)
sudo certbot certificates
# expect: VALID: 89 days

# 5. Auto-renew timer armed
systemctl list-timers | grep certbot
# expect: certbot.timer ... Mon 2026-XX-XX

# 6. Dry-run renewal works
sudo certbot renew --dry-run
# expect: Congratulations, all simulated renewals succeeded
```

## Rollback

If certbot breaks the nginx vhost:
```bash
sudo cp /etc/nginx/sites-available/janicka-shop.conf.backup \
        /etc/nginx/sites-available/janicka-shop.conf
sudo nginx -t && sudo systemctl reload nginx
# certbot auto-saves the pre-edit config to .backup — check /var/log/letsencrypt/letsencrypt.log
```

To revoke a bad cert:
```bash
sudo certbot revoke --cert-name janicka-shop.cz
sudo certbot delete --cert-name janicka-shop.cz
```
