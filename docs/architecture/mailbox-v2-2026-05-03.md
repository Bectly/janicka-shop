## 0. TL;DR

- Současný `/admin/mailbox` je placeholder: 3 Prisma modely (`EmailThread`, `EmailMessage`, `EmailAttachment`), threads list, search, compose stub. IMAP cron je naprogramovaný, ale `IMAP_*` env vars nikdy nebyly provisioned — produkční inbox je prázdný.
- Cíl: postavit v adminu **kompletní webmail klient** (Gmail/Hey/Spike úroveň, custom pro Janičku).
- **Provider doporučení**: **Cloudflare Email Routing → Resend Inbound webhook** (free, fully managed, hours-to-ship). Vlastní mailserver odmítnut (deliverability + údržba). Resend Inbound jediný = také ok, ale CF Routing dává catch-all aliases zdarma + rezervu provider switche.
- **Aliasy**: `objednavky@`, `info@`, `kontakt@`, `reklamace@`, `podpora@`, `marketing@`, `faktury@` → všechny do **shared inboxu** s alias-source labelem.
- **MVP scope**: 3-pane layout, folders (Inbox/Sent/Drafts/Trash/Archive/Spam/Starred), compose s rich editorem, reply/forward s threading, search (Postgres tsvector), R2 attachments, mobile responsive.
- **Migration phases**: A (inbound infra) → B (schema additions) → C (UI rebuild) → D (compose+outbound persist) → E (P1 features) → F (AI assist).
- **Risk register**: 10 položek (deliverability, GDPR retence, CSRF/XSS, 25MB R2 limit, Postgres cutover blocking, alias spam, CONTACT_EMAIL bug, Janička IMAP migrace).
- **Reusability**: ano pro Kryxon support inbox, viz §11.

## 2. Provider Decision (inbound mail)

### Option A — Cloudflare Email Routing → Resend Inbound webhook ⭐ DOPORUČENO

| Atribut | Hodnocení |
|---|---|
| Cost | **Free** (CF Routing zdarma; Resend Inbound v ceně Pro $20/měs už máme) |
| Complexity | Low — DNS přes CF (už na CF DNS), Routing UI klick, Resend webhook URL |
| Control | High — catch-all + aliases v CF UI, plný `raw email` payload od Resendu |
| Deliverability | Excellent — CF + Resend MTA, automatický DKIM přes Resend |
| Attachments | Resend Inbound dává parsed attachments (max 30MB total per message) |
| SPF/DKIM/DMARC | Auto přes Resend (DKIM record vygeneruje), SPF musí ručně, DMARC `p=quarantine` |
| Time-to-ship | **2–4h** (DNS prop. < 1h, webhook handler ~150 LOC, env vars) |
| Lock-in | Low — `inboundPersist()` je provider-agnostic (RFC 5322 raw → parse → DB) |

**Justification (5 řádků)**:
1. Cloudflare DNS už spravujeme (Vercel je na CF nameservers) — Email Routing je `+1 click`.
2. Resend už používáme outbound přes Pro plán — Inbound je nuceně bezplatný add-on.
3. Catch-all `*@jvsatnik.cz` zachytí všechny aliasy bez per-alias konfigurace.
4. Žádný server-side state (žádný IMAP poll, žádné failed-cron alerts) — pure webhook.
5. Resend dostává parsed mime → `inboundPersist()` se zkrátí o ~80 řádků (parser není potřeba).

### Option B — Resend Inbound bez CF (přímé MX na Resend)

- DNS: MX 10 inbound.resend.com.
- Stejně dobré deliverability, ale ztrácíme catch-all aliases (Resend Inbound má 1 endpoint per doména, ne per alias) — musíme parsovat `to:` v handleru.
- **Verdict**: ekvivalentní A bez velkých výhod. CF Routing dává jednu redundanční vrstvu zdarma.

### Option C — Vlastní mailserver (mailcow / postfix+dovecot na Hetzneru)

- Cost: ~€5/měs Hetzner CX22 + čas (10–20h setup, neustále patche).
- Deliverability: trash. New IP = blokováno gmail/seznam 4–8 týdnů (warmup). DKIM/DMARC ručně.
- Attachments: full control, ale R2 sync = vlastní kód.
- **Verdict**: ❌ Odmítnuto. Janička má jednu zaměstnankyni (sebe), je second-hand butik, ne SaaS — vlastní MTA = sebevražda.

### Doporučení
**Option A**. Implementace: BOLT separate task po schválení provider (Phase A).

### DNS plan (jvsatnik.cz)

Přes Cloudflare:
```

MX 20 linda.mx.cloudflare.net.
MX 30 amir.mx.cloudflare.net.

TXT "v=spf1 include:_spf.resend.com include:_spf.mx.cloudflare.net ~all"
```

- 7 explicit aliases → forward na `inbound@resend.com` (single Resend Inbound webhook endpoint)

## 4. Feature spec — full webmail (priorizováno)

### P0 — must-ship MVP

#### 4.1 Layout
- **Desktop (≥ 1024px)**: 3-pane (Gmail-style)
  - Left sidebar (240px): folders + labels list, "Nová zpráva" CTA top
  - Middle (380–480px): thread list, sortováno `lastMessageAt DESC`, infinite scroll (50/page)
  - Right (flex): selected thread reader, sticky reply box at bottom
- **Tablet (≥ 768px)**: 2-pane (sidebar collapsible to icons, thread list + reader)
- **Mobile (< 768px)**: single-pane stack — folder list → thread list → reader (swipe/back nav)

#### 4.2 Folders (system, ne-mazatelné)
- `Inbox` — `archived=false AND trashed=false AND folder='inbox'`
- `Hvězdičkové` — `flagged=true`
- `Odeslané` — `direction='outbound'` (bez stavu trashed)
- `Koncepty` — `Draft` model (P0, viz §5)
- `Spam` — `spamScore >= 0.7` (Resend hint) nebo manual flag
- `Archiv` — `archived=true`
- `Koš` — `trashed=true` (auto-purge po 30 dnech, GDPR-safe)

#### 4.3 Thread list item (vizuál)
```
┌────────────────────────────────────────────┐
│ ● Jana Nováková                [📎] 14:32  │
│   Reklamace bot — vrácení peněz            │
│   "Dobrý den, posílám zpět boty číslo 38…" │
│   [reklamace] [⭐]                          │
└────────────────────────────────────────────┘
```
- Bold `●` = unread (`unreadCount > 0`)
- Sender name + last message time (smart format: "14:32" / "Včera" / "23. dub")
- Subject (medium weight)
- Snippet (1 řádek, plaintext, ellipsis)
- Inline badges: alias-label barva, ⭐ flagged, 📎 attachment-count
- Hover: quick actions (archive, trash, mark-read, snooze) — keyboard `e`/`del`/`m`/`b`

#### 4.4 Compose
- **UI**: right-side drawer (Gmail-style mini-window, draggable do full-screen)
- **Editor**: TipTap (lehčí než Quill, lepší API, používá ProseMirror — ekosystém shadcn-friendly). Toolbar: Bold/Italic/Underline/Strike, lists, link, blockquote, code, heading, horizontal rule, undo/redo, "vložit signaturu", "vložit šablonu".
- Pole: To (autocomplete z `EmailMessage.fromAddress` + `Customer.email`), CC, BCC, Subject, Body, Attachments dropzone.
- Server action: `sendComposeAction()` → Resend `send` API + persist do `EmailMessage` (direction='outbound', threadId pokud reply).
- Auto-save draft každých 5s (P1, P0 = manual "Uložit koncept" tlačítko).

#### 4.5 Reply / Reply All / Forward
- Reply: prefill To = `inReplyTo.fromAddress`, Subject = `Re: {subj}`, body = `\n\nDne {date} {name} napsal:\n> {quoted}\n`.
- Reply All: To + CC z původní zprávy minus nás.
- Forward: prefill Subject = `Fwd: {subj}`, body s "----- Forwarded message -----" header.
- Outbound Message-ID: vygenerovat `<{cuid}@jvsatnik.cz>`, In-Reply-To = parent Message-ID, References = parent.References + parent.Message-ID.

#### 4.6 Threading (RFC 5322 compliant)
Already partially modeled (`messageId`, `inReplyTo`, `referencesIds`). Implementace:
1. Při příchozí zprávě: pokud `inReplyTo` matchne existující `EmailMessage.messageId` → assign na stejný `threadId`.
2. Fallback: subject normalize (`re:`/`fwd:` strip) + participant overlap → join existing thread.
3. Pokud nic = nový thread.

#### 4.7 Search (full-text)
- **Postgres tsvector** column na `EmailMessage`:
  ```sql
  ALTER TABLE EmailMessage ADD COLUMN search_vector tsvector
    GENERATED ALWAYS AS (
      setweight(to_tsvector('simple', coalesce(subject, '')), 'A') ||
      setweight(to_tsvector('simple', coalesce(fromAddress, '')), 'B') ||
      setweight(to_tsvector('simple', coalesce(toAddresses, '')), 'B') ||
      setweight(to_tsvector('simple', coalesce(bodyText, '')), 'C')
    ) STORED;
  CREATE INDEX email_message_search_idx ON EmailMessage USING GIN(search_vector);
  ```
- `simple` config (ne `czech`) — Postgres czech text search nemá oficiální dictionary, `simple` + `unaccent` extension pokryje 95% případů.
- Query: `WHERE search_vector @@ plainto_tsquery('simple', unaccent($1))`.
- **SQLite dev**: fallback na `LIKE %query%` (pomalejší, ale dev má < 100 zpráv).

#### 4.8 Read/Unread + bulk actions
- `EmailMessage.readAt` set na první view (server action, ne client-side, kvůli prefetch).
- Thread `unreadCount` = computed dotazem (snadnější, ne race-prone): `SELECT COUNT(*) FROM EmailMessage WHERE threadId=? AND readAt IS NULL`.
- Bulk: multi-select checkbox v thread list → toolbar "Označit jako přečteno / Archivovat / Smazat / Označit hvězdičkou".

#### 4.9 Attachments
- Inbound: Resend webhook posílá parsed attachments (multipart) — handler stáhne, uploaduje do R2 (`mailbox/{messageId}/{filename}`), persistuje do `EmailAttachment`.
- Render: thumbnail grid pod message body. Klik → R2 signed URL (15min TTL).
- Inline images (cid:): rewriting `<img src="cid:...">` na R2 signed URL během render.
- **Limit**: 25MB per attachment (Resend hard limit 30MB total per message).
- Viewing: PDF inline (browser native), images lightbox, ostatní download.

#### 4.10 Mobile (< 768px)
- Sticky bottom nav: Inbox / Compose / Search / Account
- Single-pane stack:
  1. Folder list (drawer behind hamburger)
  2. Thread list (default)
  3. Reader (swipe right = back, swipe left = archive)
- Compose = full-screen modal
- Reply: collapse quoted text by default

### P1 — next iteration (po MVP signoff)
- **Labels**: custom user labels, color picker, multi-label per thread, filter sidebar entry per label
- **Drafts auto-save**: debounce 5s + visible "Uloženo v {time}"
- **Snooze**: dropdown "Za 1h / Zítra ráno / Příští týden / Vlastní" → `snoozedUntil` field, cron unsnooze hourly
- **Star/Flag**: star icon, separate "Hvězdičkové" view, bulk star

- **Signatures**: per-alias default + override per send, HTML editor, image support
- **Templates**: kategorizované (order_confirmation, refund, complaint_reply, partner_outreach), variabilní substituce `{customer_name}`, `{order_id}`
- **Notification badge**: počet `unread` v admin sidebar (server action poll 30s, nebo SSE z `/api/admin/mailbox/unread-stream`)
- **Keyboard shortcuts**: react-hotkeys-hook, Gmail mapping
- **Multi-select bulk**: shift-click range select, "Vybrat vše" header

### P2 — nice-to-have
- **Multi-user inbox**: pokud Janička přijme zaměstnankyni — `EmailMessage.assignedTo`, threading "owned by" UX
- **Sentiment hint**: Manager AI worker analyzuje thread → emoji indicator (😊/😐/😠) — privacy gate (admin opt-in)
- **Auto-suggest replies**: 3 short options pod reply boxem, click → prefill (Manager AI)
- **Read receipts**: tracking pixel — **NE** (privacy, GDPR risk, neúmyslně ohrozí trust)
- **Export thread to PDF**: `puppeteer-core` server action → R2 → download

### Migration plan
- **Phase B-1** (Postgres only — po cutoveru, viz HT#45): přidat `aliasLabel`, `folder`, `snoozedUntil` na `EmailThread`. Generated column `searchVector` přes raw migration.
- **Phase B-2**: nové modely (Label, Draft, Signature, Template, Rule).
- **SQLite dev**: stejné modely, jen `searchVector` vynechat (search fallback na LIKE).

### Outbound persist (důležité)
Send action MUSÍ:
1. Vytvořit `EmailMessage` s `direction='outbound'`, vygenerovat `messageId = '<' + cuid + '@jvsatnik.cz>'`.
2. Předat ten Message-ID Resendu jako `headers: { 'Message-ID': msgId }` (Resend respektuje custom headers).
3. Update `EmailThread.lastMessageAt`, `messageCount++`.
4. Pokud reply: assign `threadId`, set `inReplyTo`, `referencesIds`.

Bez tohoto thread continuity rozbita — odpověď přijde a nepárrne se.

## 7. Performance plan

| Vrstva | Strategie |
|---|---|
| Thread list | Pagination 50/page, cursor-based (`lastMessageAt < ?`), `revalidate=30` na page.tsx (už je) |
| Body content | Lazy-load: thread list dotaz vrací jen `subject`, `snippet` (prvních 120 znaků `bodyText`), `participants`, badges. Full body fetch při open. |
| Attachments | Thumbnails on demand (R2 image proxy přes `/api/mailbox/thumb/[r2Key]`), max 200×200, cached 7d |
| Search | Postgres tsvector + GIN index (P0 prod). SQLite dev fallback LIKE. |
| Unread count | Computed via `SELECT COUNT(*) FROM EmailMessage WHERE threadId=? AND readAt IS NULL` v thread list query — index `[threadId, readAt]`. |
| Sidebar badge | SSE stream `/api/admin/mailbox/unread-stream` (P1) nebo polling 30s (P0) |
| Compose draft | Debounce 5s, server action s `revalidatePath('/admin/mailbox')` |
| Cron snooze unsnoze | Hourly `/api/cron/email-snooze-unsnooze` — `WHERE snoozedUntil < NOW()` → unset |

Cíl: thread list FCP < 800ms na desktopu Vercel CZ region.

## 8. Security

| Vektor | Opatření |
|---|---|
| XSS přes HTML body | **DOMPurify** (server-side při render) — strict allowlist tagů + `target="_blank" rel="noopener"`, strip `<script>`, event handlers, `javascript:` URLs |
| Embedded images tracking | Gmail-style: blokovat external images by default ("Zobrazit obrázky" toggle per thread) |
| Attachment exec | File-type whitelist na inbound: blokovat `.exe/.bat/.scr/.cmd/.msi/.com/.vbs/.js/.jar` → quarantine, ne persist do R2; flag thread |
| Attachment size | Max 25MB per attachment (Resend hard limit 30MB total → split across multipart) |
| Spam | (a) Resend nativní spam score → `EmailMessage.spamScore`, threshold 0.7 → folder='spam'. (b) basic regex rules (`viagra|crypto|wire transfer urgent` …) → +0.5 spamScore. |
| CSRF | Server actions už chráněné NextAuth + `useFormState`. Compose endpoint vyžaduje admin role check (viz **MEMORY: security_audit_2026_04_28** — 9 admin actions miss role check, mailbox může být v tom seznamu). |
| Rate limit compose | 100 sends/h per admin user (Upstash Redis sliding window — už existuje pattern v `/api/contact`) |
| Sender verification | Inbound webhook validace přes Resend signing (`resend-signature` header, HMAC SHA256) |
| DKIM/SPF check | Resend dává `dkim_pass`, `spf_pass`, `dmarc_pass` v payload → store v `EmailMessage.headersRaw`, render warning badge "⚠ Neověřený odesílatel" pokud false |
| Admin auth | NextAuth session, admin role check na všech `/admin/mailbox` actions (viz security audit) |
| GDPR | `EmailMessage.bodyText` může obsahovat osobní data — retence 24 měsíců, auto-purge cron, audit log při viewu (P2). Trash 30d auto-delete. Export do GDPR-export blob (P1). |

| 8 | NextAuth admin guard chybí na compose actions (viz security audit memory) | HIGH | Phase D MUST include role check (`if (session.user.role !== 'admin') throw`) na všechny `actions.ts` v `/admin/mailbox/`. |
| 9 | Resend Inbound nemá audit / replay (1 webhook attempt) | MED | Idempotent persist (`messageId` unique), Resend retries 3× při 5xx odpovědi. Pokud handler crashne → 500 → retry. Logovat failed payloads do `dev_chat_messages`-style queue pro manual replay. |
| 10 | TipTap bundle size (~80KB gzipped) → admin TTI regrese | LOW | Lazy-load editor (dynamic import) — open compose triggeruje load. Acceptable: admin panel není hot-path. |

## 11. Reusability — Kryxon support inbox

**Otázka**: lze ten webmail pattern použít i pro jiný projekt (Kryxon)?

**Ano, s constraints**:

| Sdílené | Per-projekt |
|---|---|
| Prisma core (`EmailThread/Message/Attachment/Label/Draft/Signature/Template/Rule`) | Schema instance (Kryxon má vlastní DB) |
| `inbound-persist.ts` provider-agnostic parser | Webhook URL per project (`/api/mailbox/inbound`) |
| TipTap editor component | Brand styling, signatures, templates |
| Search query layer | Domain (`kryxon.cz` vs `jvsatnik.cz`) |
| R2 attachments lib | Bucket per-project |
| DOMPurify config | — |

**Extraction path** (pokud bude Kryxon support potřeba):
- Vytvořit `@janicka/mailbox-core` (interní package, ne npm) v monorepu — ne dnes, jen mentální hranice.
- Klíčové: rozhraní `MailboxAdapter` pro DB, R2, Resend client → každý projekt si dá vlastní impl.
- 1× kód pro UI (3-pane, reader, compose) reusable přes shadcn pattern.

**Constraints**:
- DKIM/DMARC každý projekt vlastní (jiná doména).
- Aliases per-project list.
- Auth model jiný (Kryxon nemá NextAuth admin?) — adapter potřeba.
- GDPR retence policy per-projekt.

**Verdict**: ano, ale ne dnes. Janička v2 bude reference impl. Po Phase F sednout na 30min refactor: extract sdílený UI do `src/lib/mailbox-ui/` + `src/components/mailbox/` (už je v `src/components/admin/`, jen rename).
