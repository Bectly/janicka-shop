## Known caveats (Phase 1 only)

- **Auth**: NextAuth logs `UntrustedHost` for IP-based access because `NEXTAUTH_URL` is the canonical domain. Login flows will work once DNS points at the VPS ‚Äî no app changes needed.
- **TLS**: clients hitting `https://46.224.219.3` see a `geo.kryxon.cz` cert (name mismatch). Use `-k` for testing. Real cert lands in Phase 2 after DNS.
- **Hetzner Redis**: still bound to `127.0.0.1:6379` and password-protected. Vercel does NOT use this Redis (would need a tunnel). The Hetzner app instance does ‚Äî `db:ok, redis:ok` confirmed.
- **Pre-existing rsync**: a prior worker had rsync'd an x86_64 standalone bundle into `/opt/janicka-shop`. That tree was deleted before the fresh clone (mismatched native binaries for libsql/sharp on aarch64).

## Repro / rollback

Rebuild on VPS ‚Äî use the canonical deploy script (added in #937 after the
2026-04-28 13:55 ChunkLoadError outage caused by missing standalone static
sync):
```bash
ssh kryxon
sudo /opt/janicka-shop/scripts/hetzner/deploy.sh
```

The script does: `git pull` ‚Üí `npm ci` ‚Üí `npm run build` ‚Üí rsync
`.next/static/` and `public/` into `.next/standalone/` (with `--delete` on
static so stale chunks can't ChunkLoadError) ‚Üí `chown www-data` ‚Üí run
`sync-env-standalone.sh --apply` defensively ‚Üí `pm2 reload janicka-shop
--update-env`. Idempotent. Supports `--skip-pull` (use working tree) and
`--sync-only` (skip pull/install/build, only re-run the post-build sync).

Manual equivalent (do NOT use unless deploy.sh is unavailable ‚Äî it's missing
the `--delete` and the chunk-count acceptance gate):
```bash
ssh kryxon
cd /opt/janicka-shop
git pull
npm ci --no-audit --no-fund
NODE_ENV=production npm run build
rsync -a --delete .next/static/ .next/standalone/.next/static/
rsync -a public/ .next/standalone/public/
chown -R www-data:www-data .next/standalone/.next/static .next/standalone/public
cp .env.production .next/standalone/.env.production
pm2 reload janicka-shop --update-env
```

Rollback (route traffic back to Vercel ‚Äî this isn't live anyway):
```bash
ssh kryxon "pm2 stop janicka-shop"
# Vercel is still production; stopping the VPS app is harmless.
```
